Sitcom Humor Lab

Quantifying “funny” with simple, teachable math.

Project Overview

This site presents a SoME-ready MVP analyzing a sitcom’s episodes using audience ratings, joke density proxies, and sentiment (coming soon). All data loads client-side from the repository. No backend required.

Tip: If you don’t see charts, ensure the CSV paths exist in your repo (see README) or open this site via GitHub Pages.

Figure 1 — Ratings Over Time

What’s the math?
  • Smoothing: A simple moving average highlights trend over noise.
  • Normalization: Episodes are ordered by a global index across seasons.

Figure 2 — Joke Density vs Rating (Pilot)

What’s the math?
  • Proxy features: SDH “[laughter]” events/min or transcript punchline heuristics.
  • Correlation: Pearson r quantifies co-movement (not causation!).

Methods (MVP)

  1. Ratings: Pulled from OMDb/IMDb and cleaned into data/processed/imdb_ratings.csv.
  2. Joke density (pilot): Either SDH laughter tags per minute or transcript-based proxies.
  3. Join: Merge features by (season, episode), visualize trends and scatterplots.

Educational focus: smoothing, correlation, and normalization. Post‑SoME will add sentiment and character networks.
